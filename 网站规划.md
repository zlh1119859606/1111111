# 个人网站架构规划文档

> 参考风格：https://30create.usestate.org/  
> 设计理念：极简主义 × 苹果审美 × 交互表达

---

## 一、设计理念

### 1.1 核心原则
- **极简至上**：去除一切不必要的元素，只保留最核心的内容
- **留白美学**：大量留白营造呼吸感，让内容成为焦点
- **微交互**：通过细腻的动画和交互提升体验质感
- **内容为王**：设计服务于内容，而非喧宾夺主

### 1.2 视觉风格
- **色彩方案**：黑白灰为主色调，可选择性加入单一强调色
- **字体系统**：无衬线字体，大字号标题，小字号正文
- **布局方式**：居中布局，宽屏限制，垂直滚动为主
- **间距系统**：采用 8px 基础网格系统，保持视觉节奏

## 二、页面结构规划

### 2.1 整体架构
```
首页（单页应用）
├── 导航栏（固定顶部，极简）
├── 英雄区（大标题 + 副标题）
├── 内容区（可滚动展示多个区块）
│   ├── 关于我
│   ├── 作品展示
│   ├── 技能/兴趣
│   └── 联系方式
└── 页脚（版权信息）
```

### 2.2 页面区块详细规划

#### **导航栏**
- **位置**：固定顶部，滚动时保持可见
- **内容**：Logo/名称（左） + 导航链接（右）
- **样式**：透明背景，滚动后添加轻微背景模糊
- **交互**：平滑滚动到对应区块

#### **英雄区（Hero Section）**
- **布局**：全屏高度，垂直居中
- **内容**：
  - 主标题：大字号，粗体，简洁有力
  - 副标题：中等字号，描述性文字
  - 滚动提示：向下箭头或"Scroll Down"文字
- **动画**：页面加载时的淡入动画

#### **内容区块**
每个区块采用相同的设计语言：
- **标题**：左对齐或居中，大字号
- **正文**：限制宽度（最大 800px），行高舒适
- **间距**：区块之间大间距（至少 120px）
- **动画**：滚动进入视口时的淡入+上移动画

#### **页脚**
- **内容**：版权信息 + 年份
- **样式**：小字号，居中
- **间距**：与最后一个内容区块保持大间距

## 三、视觉设计规范

### 3.1 色彩系统
```css
/* 主色调 */
--color-black: #000000;        /* 纯黑 */
--color-white: #ffffff;        /* 纯白 */
--color-gray-light: #f5f5f5;   /* 浅灰背景 */
--color-gray: #666666;         /* 中灰文字 */
--color-gray-dark: #333333;    /* 深灰文字 */

/* 强调色（可选）*/
--color-accent: #007AFF;       /* 苹果蓝 */
```

### 3.2 字体系统
```css
/* 字体族 */
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", 
             "Helvetica Neue", Arial, sans-serif;

/* 字号层级 */
--font-size-hero: 64px;        /* 英雄区标题 */
--font-size-h1: 48px;          /* 一级标题 */
--font-size-h2: 32px;          /* 二级标题 */
--font-size-body: 16px;        /* 正文 */
--font-size-small: 14px;       /* 小字 */
```

### 3.3 间距系统（8px 网格）
```css
--spacing-xs: 8px;
--spacing-sm: 16px;
--spacing-md: 24px;
--spacing-lg: 32px;
--spacing-xl: 48px;
--spacing-xxl: 64px;
--spacing-hero: 120px;         /* 区块间距 */
```

### 3.4 响应式断点
```css
/* 移动端优先 */
@media (min-width: 768px) { }  /* 平板 */
@media (min-width: 1024px) { } /* 桌面 */
@media (min-width: 1440px) { } /* 大屏 */
```

## 四、交互设计

### 4.1 滚动交互
- **平滑滚动**：页面内锚点跳转使用平滑滚动
- **视口检测**：使用 Intersection Observer API 检测元素进入视口
- **滚动动画**：元素进入视口时触发淡入+上移动画（300ms 缓动）

### 4.2 导航交互
- **悬停效果**：链接悬停时轻微下划线或颜色变化
- **激活状态**：当前区块对应的导航项高亮显示
- **点击反馈**：点击时有轻微的视觉反馈

### 4.3 微动画
- **页面加载**：内容从透明到不透明的淡入（500ms）
- **滚动触发**：元素进入视口时的动画（300ms ease-out）
- **按钮交互**：悬停和点击时的轻微缩放或颜色变化

## 六、内容规划建议

### 6.1 首页内容结构
1. **英雄区**
   - 主标题：你的名字或一句话介绍
   - 副标题：简短的个人定位描述

2. **关于我**
   - 你是谁
   - 你在做什么
   - 你的兴趣和追求

3. **作品/项目**（可选）
   - 展示你的作品或项目
   - 简洁的卡片式布局

4. **联系方式**
   - 邮箱、社交媒体等
   - 简洁的链接列表

### 6.2 内容撰写原则
- **简洁有力**：每句话都要有价值
- **真实自然**：用你自己的语言，不要过度包装
- **重点突出**：重要信息用大字号或粗体强调

## 十、开发注意事项

1. **保持简单**：不要过度设计，极简是核心
2. **性能优先**：动画要流畅，不能影响性能
3. **兼容性**：确保主流浏览器正常显示
4. **可维护性**：代码要有良好注释，结构清晰
5. **可扩展性**：预留扩展空间，但不要过度设计
