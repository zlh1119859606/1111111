## 调试步骤

1. **打开浏览器开发者工具**（按 F12）
2. **查看 Console 标签**，应该看到：
   - "音频系统已初始化"
   - "找到 X 个工具项"
   - "找到 X 个项目卡片"
   - 如果有错误，会显示具体的错误信息

3. **查看 Network 标签**：
   - 筛选 "Media"
   - 查看音频文件是否尝试加载
   - 如果显示 404，说明文件不存在

4. **测试音效**：
   - 滚动到"道"章节，查看 Console 是否显示"触发章节音效"
   - 悬停在工具上，查看是否显示"工具悬停音效触发"
   - 如果显示触发但没声音，可能是文件不存在或浏览器阻止

## 常见问题

### 问题1：音效没有声音
**可能原因：**
- 音频文件不存在（最常见）
- 浏览器阻止自动播放
- 音频文件格式不正确
- 音量设置为0

**解决方法：**
1. 检查 `assets/audio/` 目录是否有音频文件
2. 检查文件名是否完全匹配（区分大小写）
3. 查看浏览器控制台的错误信息
4. 确保进行了用户交互（点击、滚动等）

### 问题2：控制台显示"音频文件加载失败"
**解决方法：**
1. 确认文件路径正确：`assets/audio/文件名.mp3`
2. 确认文件格式为 MP3
3. 检查文件是否损坏

### 问题3：音效触发但没有声音
**解决方法：**
1. 检查浏览器音量设置
2. 检查系统音量设置
3. 查看控制台是否有播放错误
4. 尝试手动点击音频控制按钮解锁音频
